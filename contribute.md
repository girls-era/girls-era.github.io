# 如何贡献

不管你是读者还是写作者，都可以参与「少女心事」。

## 留言

每个故事页面底部都有留言区。你的留言会自动出现在 GitHub Discussions 里，完全公开透明。

## 提创意

有想写的主题但还没动笔？在 GitHub 仓库的 [Issues](https://github.com/girls-era/girls-era.github.io/issues) 里提一个创意，也许会有人帮你写出来。

## 写故事

### 流程

1. Fork [仓库](https://github.com/girls-era/girls-era.github.io)
2. 在 `_stories/` 文件夹里新建一个 `.md` 文件
3. 按照下面的格式写好文件头和正文
4. 提交 Pull Request

### 文件命名

格式：`编号-标题.md`，例如 `002-体育课.md`

编号是三位数字，按顺序递增。提交前看一下现有文件，接着编号即可。

### 故事格式

```markdown
---
title: "你的故事标题"
author: "你的笔名"
date: 2026-03-30
tags: [智识, 友谊]
status: seed
---

在这里写你的故事。

可以用 Markdown 格式。段落之间空一行。

---

场景分隔可以用三条横线。
```

**关于 `status` 字段：**

- `seed`：故事种子，欢迎别人续写
- `growing`：正在生长中，有人在续写
- `complete`：已完成的故事

**常用 `tags`：**

智识、友谊、野心、贫穷、身体、竞争、家庭、成长

你也可以创建新标签。

## 续写别人的故事

看到标记为"故事种子"的故事想续写？直接在原文件的末尾添加你的段落，并在文件头加上你的名字：

```markdown
author: "原作者 & 你的笔名"
```

## 贡献者协议

提交 Pull Request 即表示你同意：

- 将出版权授予「少女心事」项目
- 保留你的署名权
- 知悉所有出版收入将全部捐赠

## 审稿标准

我们不评判文笔。审稿只关注：

- 是否真诚
- 是否尊重他人
- 格式是否正确
